# 🚀 Deployment Checklist & Documentation

## 1. Environment Requirements
- **Hosting**: Static file hosting (Firebase Hosting, Vercel, Netlify, or GitHub Pages).
- **Protocol**: Must be served via `https://` for certain browser features (though not strictly required for this app, it is best practice).
- **CORS**: Ensure the hosting provider allows fetching `.glb`, `.json`, and `.merch` files from the same origin.

## 2. Pre-Deployment Tasks
- [ ] **Assets Check**: Verify all `.glb` models (`4sh.glb`, `4fh.glb`, etc.) are in the root directory.
- [ ] **Merch Images**: Ensure `merch/mens/` and `merch/womens/` folders are populated.
- [ ] **Favicon**: Create a `favicon.png` (32x32 or 64x64) and place it in the root.
- [ ] **OG Image**: Create a `og-image.jpg` (1200x630) for social media previews.
- [ ] **Analytics**: Update the Google Analytics tag in `index.html` if using a different property.

## 3. Deployment Steps (Firebase Example)
1. Install Firebase CLI: `npm install -g firebase-tools`
2. Login: `firebase login`
3. Init: `firebase init hosting` (Select `fixtures` or root as public directory)
4. Deploy: `firebase deploy --only hosting`
